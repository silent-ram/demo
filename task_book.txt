一、主要研究内容
本课题围绕工业设备运行过程中的温度、振动、压力等实时传感器数据展开研究。主要研究内容包括：
基于 Logistic 回归的设备故障预测方法研究研究如何通过 Logistic 回归算法对工业设备的正常与故障状态特征进行建模，训练产生用于预测设备故障概率的模型，实现对设备未来可能发生故障的概率评估。
设备时序数据的采集与存储机制研究使用 InfluxDB 构建工业设备温度、振动、压力等实时采集数据的时序存储方案，研究如何在高频场景下实现高效写入与查询，为模型训练与实时预测提供数据支持。
基于 Spring Cloud 的微服务拆分设计研究采用 Spring Cloud 技术，将系统划分为数据采集模块与预警模块两部分，实现系统结构解耦、功能分离，提高系统的可维护性与扩展性。
实时故障预警机制研究基于模型的故障概率，当概率超过设定阈值时触发预警，并自动推送对应设备的历史故障维修记录，形成可用于运维决策的辅助信息。
设备故障特征趋势可视化研究使用 Python Matplotlib 实现设备特征趋势、故障概率变化等数据的可视化，使设备运行状态更直观，为故障判定提供依据。
二、主要技术指标
故障预测准确性指标Logistic 回归模型能够基于正常与故障状态数据产生有效的故障概率输出，实现对设备故障趋势的可信预测。
时序数据处理性能指标使用 InfluxDB 对温度、振动、压力等时序数据进行高效存储与快速查询，能够支持系统的实时预测与模型更新需求。
实时预警触发指标当设备故障概率超过设定阈值时，系统需能及时触发预警，并能够准确关联并推送历史故障维修信息。
可视化展示指标能够利用 Matplotlib 展示设备运行特征趋势，使用户能从图形化视角理解设备状态变化规律。
系统架构指标基于 Spring Cloud 完成数据采集模块与预警模块的微服务拆分，实现模块解耦及可拓展能力。
