# 安全配置指南

## 概述

本文档提供了工业设备故障预警系统的安全配置指南，包括环境变量配置、密钥生成方法、安全最佳实践等内容。

---

## 环境变量配置

### 必需的环境变量

系统需要以下环境变量才能正常运行。请复制 `.env.example` 文件为 `.env`，并填写实际值。

```bash
cp .env.example .env
```

### 环境变量说明

#### 数据库配置

| 变量名 | 说明 | 示例 | 安全要求 |
|--------|------|------|----------|
| `DB_USERNAME` | 数据库用户名 | `fault_warning_user` | 不要使用root用户 |
| `DB_PASSWORD` | 数据库密码 | `Str0ng@Passw0rd!` | 至少16位，包含大小写字母、数字、特殊字符 |

#### JWT配置

| 变量名 | 说明 | 示例 | 安全要求 |
|--------|------|------|----------|
| `JWT_SECRET` | JWT签名密钥 | 见下方生成方法 | 至少256位随机字符串 |

**生成JWT密钥：**

```bash
# 使用OpenSSL生成256位随机密钥
openssl rand -base64 32

# 或使用Python生成
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

#### InfluxDB配置

| 变量名 | 说明 | 示例 | 安全要求 |
|--------|------|------|----------|
| `INFLUXDB_URL` | InfluxDB服务地址 | `http://localhost:8086` | 生产环境使用HTTPS |
| `INFLUXDB_TOKEN` | InfluxDB访问令牌 | 见InfluxDB文档 | 使用最小权限原则 |
| `INFLUXDB_ORG` | 组织名称 | `your_org` | - |
| `INFLUXDB_BUCKET` | 存储桶名称 | `sensor_data` | - |

#### Flask配置

| 变量名 | 说明 | 默认值 | 安全要求 |
|--------|------|--------|----------|
| `FLASK_DEBUG` | 是否启用Debug模式 | `False` | 生产环境必须为False |
| `FLASK_HOST` | 监听地址 | `127.0.0.1` | 不要监听0.0.0.0 |
| `FLASK_PORT` | 监听端口 | `5000` | - |

#### ML服务配置

| 变量名 | 说明 | 安全要求 |
|--------|------|----------|
| `ML_API_KEY` | ML服务API密钥 | 至少32位随机字符串 |
| `ML_SERVICE_URL` | ML服务地址 | 内网地址 |

**生成ML API密钥：**

```bash
# 使用OpenSSL生成
openssl rand -hex 32

# 或使用Python生成
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

#### CORS配置

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `CORS_ALLOWED_ORIGINS` | 允许的跨域来源 | `http://localhost:3000,https://yourdomain.com` |

## 密码策略

### 数据库密码要求

- **最小长度**：16个字符
- **复杂度要求**：
  - 至少1个大写字母
  - 至少1个小写字母
  - 至少1个数字
  - 至少1个特殊字符（!@#$%^&*等）
- **禁止使用**：
  - 常见密码（password123等）
  - 字典单词
  - 连续字符（123456、abcdef）
  - 键盘模式（qwerty）

### 用户密码要求

- **最小长度**：8个字符
- **复杂度要求**：
  - 至少1个大写字母
  - 至少1个小写字母
  - 至少1个数字
- **密码历史**：不能使用最近5次使用过的密码
- **密码有效期**：建议90天更换一次

### 默认账户密码

**重要：首次部署后必须立即修改默认密码！**

- **管理员账户**：`admin` / `admin123`
- **操作员账户**：`operator` / `operator123`

修改密码方法：
1. 登录系统
2. 进入用户管理页面
3. 点击修改密码
4. 输入新密码并确认

## 密钥管理

### 密钥生成原则

1. **使用加密安全的随机数生成器**
   - OpenSSL
   - Python secrets模块
   - Java SecureRandom

2. **密钥长度要求**
   - JWT密钥：至少256位（32字节）
   - API密钥：至少256位（32字节）
   - 数据库密码：至少128位（16字节）

3. **密钥存储**
   - 使用环境变量存储
   - 生产环境使用密钥管理服务（如HashiCorp Vault、AWS KMS）
   - 绝对不要提交到版本控制系统

### 密钥轮换策略

建议定期轮换密钥：

| 密钥类型 | 轮换周期 | 轮换方法 |
|---------|---------|---------|
| JWT密钥 | 90天 | 生成新密钥，更新环境变量，重启服务 |
| API密钥 | 90天 | 生成新密钥，更新环境变量，重启服务 |
| 数据库密码 | 90天 | 在数据库中修改密码，更新环境变量，重启服务 |

**注意：轮换密钥会导致所有现有Token失效，用户需要重新登录。**

## 安全最佳实践

### 1. 访问控制

- ✅ 使用最小权限原则配置数据库用户
- ✅ 为不同环境（开发、测试、生产）使用不同的凭证
- ✅ 定期审计用户权限
- ✅ 及时删除离职员工的账户

### 2. 网络安全

- ✅ 生产环境强制使用HTTPS
- ✅ 配置防火墙，只开放必要端口
- ✅ 使用VPN或内网访问数据库和管理接口
- ✅ 禁止将敏感服务暴露到公网

### 3. 日志和监控

- ✅ 记录所有认证尝试（成功和失败）
- ✅ 记录敏感操作（用户创建、权限变更等）
- ✅ 设置异常登录告警
- ✅ 定期审查日志

### 4. 应用安全

- ✅ 保持依赖项最新
- ✅ 定期进行安全扫描
- ✅ 实施代码审查流程
- ✅ 使用静态代码分析工具

### 5. 数据保护

- ✅ 定期备份数据库
- ✅ 加密备份文件
- ✅ 测试备份恢复流程
- ✅ 敏感数据传输使用加密通道

## 应急响应流程

### 发现安全事件时的处理步骤

1. **立即响应**
   - 隔离受影响的系统
   - 保存现场证据（日志、截图等）
   - 通知安全团队和管理层

2. **事件分析**
   - 确定攻击类型和范围
   - 识别受影响的数据和系统
   - 追踪攻击来源

3. **遏制和修复**
   - 修复安全漏洞
   - 重置受影响的凭证
   - 恢复系统正常运行

4. **事后总结**
   - 编写事件报告
   - 改进安全措施
   - 更新应急响应流程
