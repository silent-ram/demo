## 1. 引言

### 1.1 编写目的

本文档描述「基于 Logistic 回归的工业设备故障预警系统」的功能需求、非功能需求及约束条件，供系统设计、开发和测试使用。

### 1.2 项目背景

在现代工业制造中，生产设备的稳定运行直接关系到企业经济效益与安全生产。设备在运行中产生的温度、振动、压力等时序数据价值日益受到重视。然而，企业在设备管理中面临以下问题：

- 设备老化特征复杂，海量高频数据难以人工准确分析；
- 传统「事后维修」或人工巡检受时空限制，无法实现全天候实时监控；
- 突发异常时无法及时获得专业预警与维修指导。

本系统旨在通过工业物联网与机器学习技术，实现设备故障的主动预警，将「事后维修」转化为「预测性维护」。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 故障概率 | Logistic 回归模型输出的设备发生故障的概率值（0~1） |
| 预警 | 故障概率超过阈值时系统自动生成的告警信息 |
| 时序数据 | 按时间顺序采集的传感器测量值（温度、振动、压力） |
| 微服务 | 独立部署、松耦合的单一职责服务单元 |
| 预测性维护 | 基于数据分析预测故障，在故障发生前安排维修的策略 |

## 2. 系统概述

### 2.1 系统目标

1. 实时采集工业设备温度、振动、压力传感器数据并持久化存储；
2. 基于 Logistic 回归算法实时预测设备故障概率；
3. 故障概率超过阈值时自动触发预警并推送至运维人员；
4. 提供设备管理、告警管理、维修记录管理等完整运维闭环；
5. 支持模型随历史数据积累进行再训练迭代。

### 2.2 用户角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 管理员（ADMIN） | 系统管理人员 | 全部功能，含用户管理、模型管理、阈值配置 |
| 运维人员（OPERATOR） | 设备运维工程师 | 设备监控、告警处理、维修记录，不含系统配置 |

### 2.3 系统边界

- **输入：** 传感器模拟数据（温度/振动/压力）、用户操作指令
- **输出：** 故障预警通知、可视化图表、维修建议
- **外部系统：** InfluxDB（时序数据存储）、MySQL（业务数据存储）

## 3. 功能需求

### 3.1 用户认证模块

**FR-01 用户登录**
- 输入用户名和密码，验证通过后签发 JWT Token（有效期 24 小时）
- 密码采用 BCrypt 加密存储
- 登录失败返回明确错误提示

**FR-02 用户注册**
- 管理员可创建新用户账号，指定角色
- 用户名不可重复

**FR-03 权限控制**
- 所有接口需携带有效 JWT Token
- 不同角色的接口访问权限按 2.2 节定义执行

### 3.2 设备管理模块

**FR-04 设备信息管理**
- 支持设备的新增、编辑、删除、查询
- 设备属性：设备编号、名称、类型、安装位置、运行状态
- 设备状态：正常（NORMAL）、预警（WARNING）、故障（FAULT）、离线（OFFLINE）

**FR-05 设备详情查看**
- 展示设备基本信息
- 展示近 1 小时实时传感器数据折线图（ECharts）
- 展示 Matplotlib 生成的多维趋势图（温度/振动/压力 + 故障概率）
- 展示该设备的历史维修记录时间轴

**FR-06 维修记录管理**
- 支持维修记录的新增、查询
- 维修记录属性：关联设备、故障描述、处理措施、维修人员、维修时间
- 处理告警时可自动创建维修记录（告警→维修闭环）

### 3.3 数据采集模块

**FR-07 传感器数据模拟**
- 每 5 秒模拟生成所有在线设备的传感器数据
- 正常状态：温度 40~60°C，振动 0.1~0.3mm/s，压力 1.0~1.5MPa，平稳波动
- 故障趋势：周期性注入异常（温度渐升至 85°C+，振动骤升至 1.0mm/s+），对应故障标签
- 数据写入 InfluxDB

**FR-08 历史数据查询**
- 支持按设备 ID 和时间范围查询 InfluxDB 历史传感器数据
- 支持最近 1h/6h/24h/7d 快捷查询

### 3.4 故障预警模块

**FR-09 实时故障预测**
- 每次数据采集后调用 ML 服务获取故障概率
- 故障概率 ≥ 阈值（默认 0.7）时自动生成告警记录
- 告警级别：低（0.7~0.8）、中（0.8~0.9）、高（0.9+）

**FR-10 实时告警推送**
- 通过 WebSocket（STOMP）将新告警实时推送至前端
- 前端监控页面无需刷新即可收到告警通知

**FR-11 告警管理**
- 告警列表支持按设备、级别、状态、时间范围筛选
- 支持分页查询
- 运维人员可将告警标记为已处理，并填写处理备注
- 处理告警时自动在 device-service 创建对应维修记录

**FR-12 告警阈值配置**
- 管理员可通过接口修改故障概率预警阈值（0~1 之间）
- 阈值持久化存储至 MySQL 配置表，重启后生效

### 3.5 机器学习模块

**FR-13 故障概率预测接口**
- 接收温度、振动、压力特征值
- 返回故障概率（0~1）和预测标签（0=正常/1=故障）

**FR-14 趋势图生成接口**
- 接收一段时间内的传感器数据序列
- 使用 Matplotlib 生成多维折线图（含故障概率曲线）
- 以 Base64 PNG 格式返回

**FR-15 模型评估指标接口**
- 返回当前模型的准确率（Accuracy）、精确率（Precision）、召回率（Recall）、F1 Score、ROC-AUC

**FR-16 模型再训练接口**
- 从 InfluxDB 拉取历史传感器数据作为训练集
- 重新训练 Logistic 回归模型并持久化保存
- 返回新模型的评估指标
- 仅 ADMIN 角色可调用

### 3.6 可视化仪表盘

**FR-17 系统首页仪表盘**
- 显示：在线设备总数、今日告警数、设备正常率、待处理告警数
- 告警级别分布饼图
- 近 24 小时告警趋势折线图
- 各设备当前故障概率排行榜

**FR-18 实时监控页面**
- 展示所有设备最新传感器数据
- 每个设备显示故障概率进度条（颜色随风险等级变化）
- 新告警通过 WebSocket 实时高亮提示

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 传感器数据写入延迟 | ≤ 100ms |
| 故障概率预测响应时间 | ≤ 500ms |
| 告警推送延迟（WebSocket） | ≤ 1s |
| 页面首屏加载时间 | ≤ 3s |
| InfluxDB 时序数据查询（1h范围） | ≤ 200ms |

### 4.2 可靠性需求

- ML 服务不可用时，alert-service 通过 Resilience4j 断路器降级，不影响主流程
- 各微服务独立部署，单个服务故障不导致整体系统崩溃
- JWT Token 过期后自动引导用户重新登录

### 4.3 安全需求

- 所有 API 请求须经 JWT 认证（登录接口除外）
- 密码 BCrypt 加密，不可逆存储
- 角色权限严格隔离，OPERATOR 无法访问管理员接口
- SQL 使用 MyBatis-Plus 参数绑定，防止 SQL 注入

### 4.4 可维护性需求

- 微服务架构，各模块独立可扩展
- 所有后端接口提供 Swagger UI 文档
- 代码遵循阿里巴巴 Java 开发规范

### 4.5 兼容性需求

- 前端兼容 Chrome 90+、Edge 90+ 现代浏览器
- 后端 JDK 17 及以上

---

## 5. 约束条件

- 毕业设计性质，使用模拟传感器数据替代真实硬件采集
- 不使用 Docker，各服务手动启动
- 模型训练使用 scikit-learn Logistic 回归，不使用深度学习框架
